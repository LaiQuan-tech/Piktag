# PikTag 實機驗收 Checklist

> 對著這張在實機上逐項測。每項都是 **動作 → 預期結果**;不符就記下該項，回報即可。
> 本批次涵蓋 2026-06-05 的修正（掃描 2 下、onboarding 按帳號判定 + `onboarding_completed`
> 欄位、登出 timeout、註冊直接走精靈、ContactSync 通知 nudge）。

## 0. 前置

- [ ] **確認在新 build**：設定 → 關於，版本 `1.0.3` + build 號是該批最新（在 commit `868707b` 之後跑出來的那支）。不是的話以下都先別測。

## A. 登出（先驗這個，它解鎖其餘測試）

- [ ] 設定 → **登出** → **可靠回到登入頁**（不卡住、不停在原畫面）。網路好應 1–2 秒內；網路慢最多 3 秒也會回（逾時會強制清 session）。

## B. 註冊 → 直接走精靈

- [ ] 登入頁 → **註冊** → 填 email（隨便填，如 `r1@test.com`）+ 密碼 → 送出
- [ ] 預期：**loading → 直接進精靈歡迎步**。中間**不閃首頁、不跳「註冊成功」彈窗**。

## C. 精靈逐步（核心）

- [ ] **進度指示**：三段進度條，跟著步驟前進
- [ ] **Step 1**：頭照（可略過）、名字、username
  - [ ] username **即時查重**：打一個一定被佔的（如 `armand`）→ 顯示不可用；打沒人用的 → 顯示可用；**未通過不能下一步**
- [ ] **返回**：第 2、3 步左上返回鍵能回上一步，資料還在
- [ ] **Step 2**：headline、bio、標籤
  - [ ] 標籤**即時寫入**：加一個標籤馬上變 chip；**少於 3 個不能下一步**
- [ ] **Step 3**：連結 ≥3（含電話 / email）
  - [ ] 連結 picker 依語言出對的快捷（中文看到 LINE / 微信 / 支付寶；英文看到 WhatsApp / PayPal 等）
  - [ ] **少於 3 條不能完成**
  - [ ] **ContactSync 軟詢問**：若這步有「從通訊錄找朋友」入口 → 先出**自家說明卡**，按了才跳**系統權限**（不是一進來就跳系統視窗）
- [ ] **情境說明**：每步都有一句說明它為什麼存在（不是空白表單）
- [ ] 走完 → 慶祝動畫 → 進 App

## D. 按帳號判定（這次修的重點，務必測）

- [ ] **空帳號會走精靈**：登入 `piktag.tester01@gmail.com`（密碼 `Piktag2026!test`）→ **精靈出現**（它 profile 是空的）
- [ ] **走完不再問**：把 tester01 精靈走完 → 殺掉 App 重開 → **直接進首頁，不再跳精靈**
- [ ] **step 1 中途離開會重問**：登入 `tester02` → 只填完 Step 1（名字 + username）→ 殺掉 App → 重開 → **精靈再次出現**（不會因為有名字就放行）
- [ ] **既有完整帳號不被逼**：登入原本那個有 3 個好友的帳號 → **直接進首頁，不會被抓去走精靈**

## E. 名片掃描（2 下，點 icon 直接鏡頭）

- [ ] 好友頁右上 **+人 → 立刻是相機 live 畫面**（一段轉場，不先閃一個空白表單）
- [ ] 按**快門** → 進到**已預填**的聯絡人表單
- [ ] 相機上的「**或手動輸入**」→ 進**空白**表單、游標在姓名
- [ ] 相機左上 **X** → 回好友頁

## F. 兩帳號互動（選測，需兩支裝置或兩帳號交替）

- [ ] tester01、tester02 各登一支 → 互掃 QR / 互加好友 → 兩邊好友列表都看到對方
- [ ] 兩邊都標重疊概念（如都 #咖啡 #登山）→ 搜尋一邊的概念，確認另一邊被配對到

## ⚠️ 不能即時測的一項（說明，不是 bug）

- **通知「從通訊錄找朋友」**（`contact_sync_nudge`）：每日 **cron**（台北 01:30）推送，只發給**註冊滿 1 天、好友 < 5、沒收過**的帳號。
  - `tester01/02`（今天才建）今天**還不符合**，要等隔天。
  - 原本那個 3 好友帳號**符合**，可在隔天早上看鈴鐺。
  - 想從 server 端確認它有正確進到某帳號的鈴鐺，再跟我說。
