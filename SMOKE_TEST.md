# PikTag 送審前煙霧測試(手動,約 10 分鐘)

> 每次送商店審核前,拿實機(TestFlight / 內測軌最新 build)走完這一輪。
> 任何一步不符預期 = 不送審,先修。
> 需要:兩台實機(或一台 + 一位朋友)、一張實體名片。

## 前置確認(1 分鐘,電腦上)

- [ ] `cd mobile && npx tsc --noEmit` 乾淨
- [ ] GitHub Actions:iOS / Android / Supabase Deploy 最新一次全綠
- [ ] Sentry:最近 24h 無新增 crash(crash-free rate > 99%)

## 主迴圈(照順序走)

1. **註冊 → 精靈**:用新 email 註冊 → 應**直接**進三步精靈(無「註冊成功」彈窗、無空首頁閃現)。
   走完 身分 → 標籤(試 AI 推薦)→ 連結 → 完成動畫 → payoff 頁 → 進主畫面。
2. **官方帳號**:好友列表應已有 @piktag;通知鈴應有一則「成為好友」。
3. **掃名片(單人價值)**:好友頁空狀態點「掃一張名片試試」→ 拍實體名片 →
   **欄位須立即出現**(辨識不能等)→ AI 標籤 1–2 秒後以預選紫 chip 入列 → 儲存成功。
4. **互掃 QR(北極星)**:兩台機互掃 → 加好友成功 → 對方出現在好友列表。
   **推播權限彈窗應在此刻(或首次開通知頁)出現 —— 不是 App 啟動時。**
5. **搜尋**:搜一個標籤(含跨語言,例:搜 cat 應命中 #養貓)→ 點結果 → 進好友詳情。
6. **聊天 + 推播路由**:發一則訊息 → 另一台**殺掉 App** → 收到推播 → 點開 →
   **必須直接落在該對話**(落在搜尋頁 = 舊 bug 回歸,擋審)。
7. **通知 tab**:三個分頁都有內容歸類、無空白卡;icon badge 數字與未讀一致;點通知能正確跳轉。
8. **Dark mode**:Settings 切換 → 快掃 好友/搜尋/聊天/通知/個人 五頁,找「白底白字/黑底黑字」。
9. **刪帳號防復活**:Settings 刪除帳號 → 用**同一個 email** 重新註冊 →
   舊好友/標籤**不得**自動回來(回來 = resurrection bug,擋審)。
10. **語言**:切到 English → 快掃主要頁面,無中文殘留、無 key 名直接顯示。
